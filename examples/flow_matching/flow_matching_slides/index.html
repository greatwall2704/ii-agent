From Normalizing Flows to CNFs

Normalizing Flows

Transform simple distribution (e.g., Gaussian) to complex data distribution through invertible transformations.

$z_0 \xrightarrow{f_1} z_1 \xrightarrow{f_2} z_2 \xrightarrow{f_3} x$

Continuous Normalizing Flows

Replace discrete transformations with continuous ODEs:

$$\frac{d\mathbf{z}_t}{dt} = v(\mathbf{z}_t, t)$$

where $v(\mathbf{z}_t, t)$ is a learned vector field

Flow Matching: Core Concepts

The Problem

Training CNFs traditionally requires:

  • ‚Ä¢ Expensive ODE simulation
  • ‚Ä¢ Complex likelihood calculations
  • ‚Ä¢ Unstable training dynamics

The Solution

Flow Matching provides:

  • ‚Ä¢ Simulation-free training
  • ‚Ä¢ Direct vector field regression
  • ‚Ä¢ Stable optimization

Flow Matching Objective

$$\mathcal{L}_{\text{FM}}(\theta) = \mathbb{E}_{t, p_t(x)} \left[ \|v_t(x) - u_t(x)\|^2 \right]$$

Learn vector field $v_t(x)$ to match target vector field $u_t(x)$

Mathematical Framework: Three Key Theorems

Theorem 1: Marginal Construction

Conditional vector fields $u_t(x|x_1)$ can generate marginal probability paths $p_t(x)$:

$$u_t(x) = \int u_t(x|x_1) \frac{p_t(x|x_1)q(x_1)}{p_t(x)} dx_1$$

Theorem 2: Loss Equivalence

Conditional Flow Matching loss equals Flow Matching loss (same gradients):

$$\nabla_\theta \mathcal{L}_{\text{FM}}(\theta) = \nabla_\theta \mathcal{L}_{\text{CFM}}(\theta)$$

Theorem 3: Gaussian Paths

For Gaussian conditional paths, the vector field has explicit form:

$$u_t(x|x_1) = \frac{\sigma'_t(x_1)}{\sigma_t(x_1)}(x - \mu_t(x_1)) + \mu'_t(x_1)$$

Conditional Flow Matching (CFM)

Key Insight

Instead of learning the complex marginal vector field $u_t(x)$, learn simpler conditional vector fields $u_t(x|x_1)$.

CFM Training Loss

$$\mathcal{L}_{\text{CFM}}(\theta) = \mathbb{E}_{t,q(x_1),p_t(x|x_1)} \left[ \|v_t(x) - u_t(x|x_1)\|^2 \right]$$
$t \sim \text{Uniform}[0,1]$ - Random time
$x_1 \sim q(x_1)$ - Data sample
$x \sim p_t(x|x_1)$ - Conditional path

Training Process

1 Sample time $t$ and data $x_1$
2 Sample from conditional path $p_t(x|x_1)$
3 Compute target vector field $u_t(x|x_1)$
4 Train model $v_t(x)$ to match target

Probability Paths & Vector Fields

Different choices of conditional probability paths lead to different training dynamics:

Variance Exploding

$$p_t(x|x_1) = \mathcal{N}(x|x_1, \sigma_{1-t}^2 I)$$
  • ‚Ä¢ Data preserved, noise added
  • ‚Ä¢ Explores wide space
  • ‚Ä¢ Good for diversity

Variance Preserving

$$p_t(x|x_1) = \mathcal{N}(x|\alpha_{1-t}x_1, (1-\alpha_{1-t}^2)I)$$
  • ‚Ä¢ Similar to DDPM
  • ‚Ä¢ Stable variance
  • ‚Ä¢ Well-studied properties

Optimal Transport

$$p_t(x|x_1) = \mathcal{N}(x|tx_1, (1-(1-\sigma_{\min})t)^2I)$$
  • ‚Ä¢ Straight-line paths
  • ‚Ä¢ Minimal transport cost
  • ‚Ä¢ Fastest sampling

Key Properties

Boundary Conditions:
  • ‚Ä¢ $t=0$: $p_0(x|x_1) = \mathcal{N}(0, I)$ (noise)
  • ‚Ä¢ $t=1$: $p_1(x|x_1) \approx \delta(x - x_1)$ (data)
Vector Field:
  • ‚Ä¢ Defines how samples flow over time
  • ‚Ä¢ Learned by neural network
  • ‚Ä¢ Enables generation via ODE solving

Training & Sampling Process

Training Phase

1. Data Preparation

Sample time $t \sim U[0,1]$ and data point $x_1 \sim q(x_1)$

2. Path Sampling

Sample $x_0 \sim \mathcal{N}(0,I)$ and compute $x_t$ using probability path

3. Target Computation

Calculate target vector field $u_t(x_t|x_1)$ analytically

4. Model Update

Train neural network $v_t(x_t)$ to match $u_t(x_t|x_1)$

Training Loss

$$\mathcal{L} = \mathbb{E}[\|v_t(x_t) - u_t(x_t|x_1)\|^2]$$

Sampling Phase

1. Initialize

Start with noise sample $x_0 \sim \mathcal{N}(0,I)$

2. ODE Integration

Solve $\frac{dx_t}{dt} = v_t(x_t)$ from $t=0$ to $t=1$

3. Numerical Solver

Use Euler, RK4, or adaptive solvers

4. Generated Sample

Final $x_1$ is the generated data sample

Key Advantage

No simulation during training!
Only need ODE solving for generation
